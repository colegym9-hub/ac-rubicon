# Vision

> What we're building and what "done" looks like. Claude reads this before any work.

---

## What This Project Is

AC Rubicon — a personal life-OS that unifies Cole's **second brain** (his LLM-maintained wiki of everything he reads, watches, and thinks) with a **command center** (daily planning, tasks, tracking). A phone-installable Next.js PWA + Supabase that is the **source of truth for projects, tasks, daily tracking, AND the second brain itself**. The brain is now a **first-class surface inside the app, hosted in Supabase** — Cole captures sources (links, voice, photos, notes) into it, an automated pipeline files them into a wiki, and he can browse, edit, and chat with that knowledge. The brain informs the AI day-planner; the nightly recap feeds the brain. Tagline: "optimizing my past into building my future."

---

## Who It's For

Cole — a solo student and entrepreneur (A.C Media photography business, content creation, school, health/WHOOP). Balances deep work, light work, gym, friends, going out. Usually not at his computer until later in the day, so the daily plan must reach and be adjustable from his **phone**. Already journals a nightly recap every single day, and already collects sources into a markdown brain — both habits this app makes first-class.

## What "Done" Looks Like — the four tabs

A phone-installable PWA with **four tabs**:

- **Today** — the calendar timeline is the primary view; each block has an inline check-off; a morning check-in pops up once a day (fill out yesterday's log if it's missing, or review it if it's done); the daily log (tracking metrics + recap) lives behind a persistent "Log today" button; "re-plan from now" asks two quick questions and regenerates the rest of the day. The day is generated by the AI planner.
- **Projects** — projects grouped by category, priority-weighted tasks, a one-off Inbox; fast inline add, check-off, priority. (Unchanged from v1.)
- **Insights** — task throughput, tracking trends, and the planned-vs-done adherence mirror, plus a weekly AI-generated signal card. (Was "Graphs.")
- **Brain** — capture (link / voice / photo / note → markdown → raw source → auto-filed into the wiki), browse the wiki by domain, read and edit pages, and "ask your brain anything" chat over the wiki + live app data.

Projects/tasks/tracking/plans **and** the brain (raw sources + wiki) live in Supabase. Tracking is logged from Today; metrics are managed from a lightweight Settings route + the command palette.

---

## How the AI Runs (the cost model)

All AI work runs as **Claude Code cloud routines** on Cole's existing subscription — fired on demand by the app (a wakeup signal; the actual request lives in a Supabase table) or on a schedule. This covers the daily planner, brain ingest (raw source → wiki, following the brain's `CLAUDE.md` rules), brain chat, the two-question re-plan, and the weekly lint + insight. **No pay-per-token Claude API key, and no OpenAI key.** Retrieval uses Postgres full-text search, not vector embeddings. The only external keys are conversion services on free tiers: **Supadata** (YouTube / Instagram / TikTok / etc. transcripts) and **Deepgram** (voice); images and handwriting are read by Claude vision inside the routine.

## What Success Means

Cole actually **follows** the plan (the adherence chart shows it) and manages his real projects/tasks here daily instead of scattered lists — fixing the exact failure of the old hard-coded command center, which he "didn't follow well." And the brain **compounds**: sources go in from the phone, the wiki grows itself, and Cole gets value back out through chat and a planner that knows his own context.

## Constraints

- Claude builds; Cole is a non-dev who lives in the tool and makes the product calls.
- Stay on **Supabase free tier + Vercel hobby tier**. The AI runs on Cole's Claude subscription (≈ **$0 marginal API**). External services (Supadata / Deepgram) on free tiers.
- **Phone-first** (PWA before any native app); all mockups phone-framed.
- The brain **moves to the cloud (Supabase)**: the existing ~1,244 raw sources + 29 wiki pages migrate in as the seed; ongoing capture feeds it; the local files become a cold backup. *(This supersedes the old "brain stays as-is / no re-ingest now" constraint.)*
- Reuse the existing dark OKLCH glass/teal design tokens from the old Electron prototype.
- **DO NOT build a Notion clone:** no general block editor, no user-definable DB engine, no real-time collaboration, no offline-sync engine. The Brain wiki is Claude-maintained markdown over a fixed schema — not a block editor.

## Out of Scope (For Now)

- Native iPhone app — only if PWA limits actually bite.
- Multi-user; payments; real-time collaboration.
- Vector / semantic search — full-text search is the v1 retrieval; revisit only if recall proves weak (Supabase has a free in-DB embedding option before any OpenAI dependency).
- Instagram beyond what Supadata returns; background (screen-off) voice recording (an iOS PWA limit).
- AI "coach" / long-horizon weekly reviews beyond the weekly insight card — later, once brain history accrues.
